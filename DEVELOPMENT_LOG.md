# AITeach 开发日志

更新时间: 2026-04-18

## 阶段 0: 课程知识库准备

- 整理电机学课程问答数据，建立 `knowledge_base/` 本地知识库。
- 明确项目不只做通用聊天，而是做“电机学场景化学习系统”。

## 阶段 1: Streamlit 原型期

- 使用 Streamlit 快速搭出最初版本。
- 功能包含图片解题、文字问答、基础仿真和错题记录。
- 这一阶段重点是验证“功能能不能串起来”，不是工程化。

## 阶段 2: AI 解题与 RAG 强化

- 接入 Qwen 文本模型与视觉模型。
- 建立 OCR 提题 + 知识库召回 + 解题 Prompt 的基本链路。
- 增加课程参考内容拼接，降低大模型在电机学场景中的胡言乱语概率。

## 阶段 3: Web 化重构

- 项目主入口从 Streamlit 转向 `FastAPI + 原生 Web 前端`。
- 建立 `api/`、`services/`、`simulation/`、`storage/` 的分层结构。
- 前端改为单页 Web 控制台，支持聊天式 AI 解题、错题本、仿真实验和知识问答。

## 阶段 4: 错题本闭环完善

- SQLite 错题本改为可分页、可编辑、可分析。
- 加入 AI 错题教练，支持学情分析、出题练习和解析生成。
- 优化保存反馈和局部刷新，减少页面跳动。

## 阶段 5: 仿真模块扩展

- 固定出四组电机学基础实验。
- 仿真结果支持关键指标、数据预览和实验建议。
- 为异步电机等实验补充了图形化输出链路，仿真后端不仅返回数字，还能返回可直接在前端显示的曲线图片数据。
- MATLAB 侧建立三层策略:
  - Engine 优先
  - CLI 兜底
  - Python 回退保底
- 这一阶段的重要亮点是: MATLAB CLI 已经可以参与生成仿真图像，前端也预留了图像展示位，使“仿真结果可视化”成为系统卖点之一。

## 阶段 6: 前端体验整理

- 页面整体改成真正 Web 形态，不再依赖 Streamlit 界面。
- AI 解题改成聊天框形态，支持上下文追问、附件上传和流式回答。
- 错题本、仿真页、选择器、提示条、按钮动效等 UI 做过多轮整理。

## 阶段 7: 2026-04-18 收尾整理

### 7.1 AI 解题流式公式渲染修复

- 为流式回答增加公式片段保护:
  - `protectMathSegments(...)`
  - `restoreMathSegments(...)`
- 增加 MathJax 节流调度:
  - `scheduleTypesetMathJax(...)`
- 流式阶段先判断公式分隔符是否闭合，再触发排版，避免一边输出一边把公式结构打碎。
- 解决了“流式输出时公式只显示原始文本、结束后才突然变样”的问题。

### 7.2 流式链路稳定性补强

- `/api/solve/chat/stream` 现在会稳定发出 `done` 事件。
- 前端新增 `receivedDone` 兜底逻辑，旧链路或异常中断时也能尽量收尾。
- 修复了流式生成后临时图片文件未及时清理的问题。
- 前端对 `payload.error` 与 `payload.message` 都做兼容处理。
- 为减少流式阶段的跳闪，前端开始按帧合并聊天重绘，而不是每个增量都强制整块刷新。

### 7.3 错题闭环补强

- AI 教练出的练习题在用户点击“查看解析”后，会出现单独的“加入错题本”选项，由用户自己决定是否沉淀进错题本。
- 新增了练习题到错题本的格式化转换逻辑，会把题目、考点、提示、参考答案、AI 解析和用户作答整合成可复盘记录。
- 通过本地状态位与按钮状态联动，避免同一道 AI 练习题被重复加入错题本，同时保留用户主动控制权。

### 7.4 分发与启动体验优化

- `requirements.txt` 改为固定版本，提升可复现性。
- `start_aiteach.ps1` 改为:
  - 支持 `python` 或 `py` 启动虚拟环境创建
  - 基于 `requirements.txt` 哈希跳过重复安装
  - 继续保留一键启动浏览器体验
- `.env.example` 增补可选模型配置项，便于同学自助配置。

### 7.5 文档整理

- 重写 `CURRENT_ARCHITECTURE.md`，把分层、调用链、流式公式渲染、仿真图片输出与稳定性策略单独讲清楚。
- 将分发策略单独放到 `CLASSMATE_DISTRIBUTION_PLAN.md`，不再混进架构说明。
- 本轮完成后，项目已经具备“本地一键启动 + 依赖固定 + MATLAB 可选 + AI Key 自填”的稳定分发形态。
- 前端代码开始从单一大脚本向功能脚本拆分，当前已整理出 `app.js`、`app_coach.js`、`app_boot.js` 三层结构，为后续继续细化解题页、仿真页和共享渲染逻辑打下基础。
