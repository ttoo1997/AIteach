# AITeach 稳定分发指南

更新时间: 2026-04-18

## 1. 目标

这份方案的目标不是把 AITeach 直接做成公网 SaaS，而是把它整理成一个:

- 同学几乎不用懂编程
- 拿到压缩包后可以尽快跑起来
- 遇到 MATLAB 缺失也不会直接报废
- 适合课程展示、班级内部试用和答辩验收

的“高完成度本地分发版”。

这里强调的“稳定分发”，核心不是功能最多，而是:

- 同学拿到后能稳定启动
- 环境差异不会轻易把项目跑崩
- 即使没有 MATLAB，也能继续使用主要功能
- 你后续维护成本可控

## 2. 最推荐的分发路线

### 路线 A: 班级内部一键启动版

这是当前最推荐的版本。

你分发给同学的是:

- 整个项目文件夹
- `.env.example`
- `start_aiteach.bat`
- `stop_aiteach.bat`

同学只需要:

1. 安装 Python 3.10+
2. 双击 `start_aiteach.bat`
3. 第一次启动时填写 `.env.local` 里的 `DASHSCOPE_API_KEY`
4. 等待浏览器自动打开

优点:

- 成本最低
- 你当前就能做到
- 维护最轻
- 对答辩和同学体验都足够好

限制:

- 首次运行依赖本机有 Python
- AI Key 需要自己填
- MATLAB 不是人人都有

### 路线 B: 绿色便携版

如果你想进一步降低门槛，可以做成“自带运行时”的压缩包。

目标结构建议是:

```text
AITeach-release/
  app/
  runtime/
  wheels/
  start_aiteach.bat
  stop_aiteach.bat
  .env.example
```

思路是:

1. `runtime/` 放入 Windows embeddable Python
2. `wheels/` 预下载依赖包
3. 启动脚本优先使用本地 `runtime/python.exe`
4. 不再要求同学先装 Python

优点:

- 对不会编程的同学最友好
- 可真正做到“解压即用”

限制:

- 你需要额外准备嵌入式 Python 和离线 wheel 包
- 包体积会明显变大

## 3. MATLAB 该怎么处理

### 3.1 当前最稳策略

不要把 MATLAB 作为“必须依赖”。

保持现在的策略:

- 有 MATLAB: 允许走 Engine / CLI
- 没 MATLAB: 自动回退到 Python 仿真

这样同学端不会因为没有 MATLAB 就完全无法使用。

### 3.2 对同学的实际提示

建议你在分发说明里明确写:

- 仿真模块默认可直接使用
- MATLAB 是增强功能，不是必需项
- 如果本机安装了 MATLAB，可以在页面里手动指定 CLI 路径

## 4. AI Key 怎么分发

### 4.1 最安全的做法

让每位同学自己填写 `DASHSCOPE_API_KEY`。

理由:

- 不泄露你的主 Key
- 不会因为多人共用把额度迅速打空
- 后续责任边界更清楚

### 4.2 演示版做法

如果只是短期课堂展示，也可以准备一个临时 Key 放在你的演示机上，但不要把带真实 Key 的 `.env.local` 直接打包发给别人。

## 5. 我建议你实际采用的稳定分发方案

如果以你当前阶段的时间和收益比来判断，我建议你采用下面这套组合:

1. 当前仓库整理为“班级内部一键启动版”
2. 依赖固定到稳定版本
3. 启动脚本支持自动创建虚拟环境、按哈希跳过重复安装
4. MATLAB 保持可选
5. `.env.example` 作为模板，不分发真实 Key
6. README 只写“如何运行”
7. 这份文档专门写“如何给同学发”

这套方案已经足够接近你当前阶段的“稳定分发”目标:

- 对你维护成本低
- 对同学门槛低
- 对答辩叙事也合理

## 6. 你真正发给同学时的操作清单

### 6.1 你这边先准备

1. 确认 `.env.local` 里没有你的真实 Key
2. 保留 `.env.example`
3. 删除运行时日志和临时文件
4. 确认 `start_aiteach.bat` 可以一键启动
5. 打包成 `AITeach_classroom_release.zip`

### 6.2 同学拿到后

1. 解压缩
2. 双击 `start_aiteach.bat`
3. 若提示缺 Python，则先安装 Python 3.10+
4. 按提示补 `.env.local`
5. 浏览器自动打开后开始使用

## 7. 如果以后真的要更进一步

如果你后续要做“更正式的产品化分发”，顺序建议是:

1. 先做绿色便携版
2. 再做局域网部署版
3. 最后才考虑公网部署

不要一上来就做公网服务器，因为那会引入:

- 域名
- HTTPS
- 备案
- 成本
- 并发
- AI 调用额度
- 数据安全
- MATLAB 服务部署

这些问题和你当前的比赛目标并不完全一致。

## 8. 一句话结论

当前最好的“完美分发”不是公网发布，而是:

`本地一键启动 + 依赖锁定 + MATLAB 可选 + Key 自填`

这条路线最稳，也最适合你现在的项目阶段。
